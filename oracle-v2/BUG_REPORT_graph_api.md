## Summary

Graph API `discoverRelationships()` claims to create 12,952 relationships but data is not persisted. All graph queries return empty results.

```bash

# All queries return empty
GET /api/graph/top-concepts → {"concepts":[],"count":0}
GET /api/graph/concepts/oracle/related → {"related":[],"count":0}
```

---

## Root Cause Analysis

### Primary Issue: Missing Table in Runtime Schema

**File:** `src/db/runtime-schema.ts`

The `concept_relationships` table is defined in:
- `src/db/schema.ts` ✅ (Drizzle schema)
- `scripts/migrate-p4-graph.ts` ✅ (Manual migration)

But NOT in:
- `src/db/runtime-schema.ts` ❌ (Auto-created on server start)

**Impact:** When server starts, `ensureSchema()` creates all tables EXCEPT `concept_relationships`. All INSERT operations fail silently.

### Secondary Issue: Silent Failure in Service

**File:** `src/knowledge-graph-service.ts`

```typescript
// Line 80-82: Errors logged but not thrown
} catch (error) {
  logNonFatal('graph.record_relationship', error, { rel });
}
```

The service catches database errors and logs them as warnings, but continues execution and returns "success" anyway.

### 1. Add Table to Runtime Schema

  // Concept Relationships (P4 Knowledge Graph)
  database.exec(`
    CREATE TABLE IF NOT EXISTS concept_relationships (

      from_concept TEXT NOT NULL,
      to_concept TEXT NOT NULL,
      relationship_type TEXT NOT NULL,
      strength INTEGER DEFAULT 1,
      last_seen INTEGER NOT NULL,
      created_at INTEGER NOT NULL,
